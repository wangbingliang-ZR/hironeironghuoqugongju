# DEV WORKFLOW

## 开发规则

### 每次只开发一个闭环
1. 先完成一个功能模块
2. 确保可运行
3. 再开发下一个

### 代码规范
- 不主动增加依赖
- 不擅自优化未来
- 优先真实运行

### 思考原则
每次生成代码前思考：这是不是过度工程？
如果是：禁止实现
