# HIRO 产品总开发说明书

## 1. 项目一句话定位

HIRO 不是一个普通的聊天机器人，也不是一个励志情绪陪伴工具。

HIRO 要做的是：

> 让一个在职场、关系、生活压力里说不出口的人，把那句话丢出来，然后被接住、被整理、被看见，并最终获得一个更清楚的下一步。

外部获客工具负责让人停下来，HIRO 产品本体负责把人接住。

整个项目的核心不是“劝人变好”，而是：

> 不上课，不安慰，不喊口号。  
> 先把那句话说出来。

---

## 2. 项目整体结构

当前项目未来会形成三层系统：

```text
第一层：爬虫获客系统
主动去各个平台寻找潜在用户、痛点话题、真实表达。

第二层：内容钩子系统
把真实痛点变成高共鸣内容，发布到小红书、知乎、公众号、短视频等平台。

第三层：HIRO 产品本体
承接被内容打动的人，让他把自己的真实情况说出来，并逐步转化为付费用户。
```

这三层不是三个孤立项目，而是一条完整链路：

```text
真实痛点采集 → 内容钩子生产 → 平台分发 → 评论/私信互动 → HIRO 承接 → 情绪梳理/回应生成 → 付费转化
```

---

## 3. 爬虫获客系统负责什么

### 3.1 核心作用

爬虫系统不是为了“群发广告”，也不是为了粗暴收集用户。

它的核心作用是：

1. 找到真实用户正在说的话
2. 发现平台上正在高频出现的痛点
3. 收集自然语言表达方式
4. 反哺内容选题和 HIRO 产品语料
5. 辅助识别潜在用户所在平台和场景

### 3.2 应该采集什么

优先采集这些内容：

- 职场委屈
- 面试焦虑
- 领导压迫
- 同事关系
- 试用期不安
- 裸辞后迷茫
- 讨好型人格
- 不敢拒绝
- 被否定后的自我怀疑
- 加班疲惫
- 工作没有意义感
- 想离职但不敢

重点不是采集“关键词”，而是采集原话。

例如：

```text
“领导说我不够主动，可我已经连续加班两周了。”

“我每次想拒绝，话到嘴边就变成了好。”

“我明明没做错，但他一开会点我名字，我就开始慌。”
```

这些原话未来会成为：

- 内容选题
- hook素材
- 评论引导素材
- HIRO 输入示例
- 付费功能里的场景模板

### 3.3 爬虫系统不应该做什么

爬虫系统不应该：

- 批量骚扰用户
- 自动私信推销
- 大规模硬广触达
- 把用户当作线索表格处理
- 用爬来的内容直接冒充原创

爬虫系统的价值是“理解真实世界正在发生什么”，不是低质量营销。

## 4. 内容钩子系统负责什么

### 4.1 内容钩子的定位

内容钩子系统负责把真实痛点变成平台内容。

它不是鸡汤号，不是职场导师号，也不是情绪安慰号。

它的定位是：

> 替普通人把心里话说出来。

用户刷到内容时，第一反应应该是：

```text
这说的不就是我吗？
```

而不是：

```text
这个人在教育我。
这个人在安慰我。
这个像AI写的。
这个又在卖东西。
```

### 4.2 当前内容系统的核心能力

当前内容生成系统已经形成几项原则：

#### 1. 双模式写作

```text
模式A：场景叙事型
适合试用期、面试、裸辞、深夜加班、不敢拒绝等具体场景。

模式B：观点爆款型
适合领导画饼、越能干越被塞活、讨好型人格、职场内耗等观点类话题。
```

#### 2. 模式A规则

模式A必须像一个人在讲刚发生的事：

- 有动作
- 有物体
- 有时间地点
- 有身体反应
- 不插入分析句
- 不给道理
- 不解释为什么

错误示例：

```text
试用期是双向选择，不是单方面审判。
```

这种句子虽然听起来正确，但有“道理味”，会破坏真实感。

正确方向：

```text
他把你叫进会议室，门一关，你手里攥着笔帽，指甲都掐进去了。
```

#### 3. 模式B规则

模式B要短、狠、具体：

```text
不是你能干，是别人知道你好用。
越能干，活越往你桌上堆。
键盘敲得再快，锅还是你的。
```

模式B不能软成安慰，也不能变成人生总结。

结尾必须像“钉一下”，最好落到具体东西上：

- 工资
- 工牌
- 会议
- 消息
- PPT
- 电脑
- 键盘
- 饭点
- 地铁
- 灯

#### 4. 禁止鸡汤和AI味

内容中坚决避免：

```text
没关系
会好的
别焦虑
慢慢来
都会过去
你值得
相信自己
治愈
和解
自洽
内核
停止内耗
人间清醒
```

HIRO 的内容风格必须记住一句话：

> 我不是来安慰你，我是来把这件事说清楚。

## 5. 评论引导系统负责什么

### 5.1 评论不是互动装饰，而是第二个钩子

内容让用户停下来。

评论引导让用户开口。

用户只点赞收藏还不够，真正有价值的是他说出自己的情况。

所以评论引导不能问：

```text
你怎么看？
你认同吗？
你有类似经历吗？
欢迎分享你的故事。
```

这些都太泛泛，用户不一定会说。

评论引导应该问一个只有经历过的人才答得出来的问题。

例如：

```text
你上次想说“不”，后来为什么又说了“好”？

你被约谈那天，第一句话听到的是什么？

你加班最晚那次，办公室还剩几盏灯？
```

### 5.2 经典评论引导库

当前项目已沉淀 18 条经典评论引导：

```text
1. 你上次想说'不'，后来为什么又说了'好'？
2. 你被约谈那天，第一句话听到的是什么？
3. 你加班最晚那次，办公室还剩几盏灯？
4. 你最近一次被塞活，是因为什么没拒绝？
5. 你被领导当众否定那次，手里拿着什么？
6. 你最想辞职的那天，是几点下的班？
7. 你上次被画饼，对方原话怎么说的？
8. 你最不想回的那条消息，最后回了什么？
9. 你面试前一晚，反复看的是简历还是聊天记录？
10. 你最累那天，电脑关了几次又打开？
11. 你第一次觉得这份工作不对劲，是哪一个瞬间？
12. 你哪次明明委屈，还是先说了'没事'？
13. 你最近一次在工位上发呆，是因为哪件事？
14. 你最想把哪句话原封不动丢出来？
15. 你哪次下班路上，突然一句话都不想说？
16. 你最近一次忍住没发出去的消息，写了什么？
17. 你哪次觉得自己像在替整个组兜底？
18. 你最怕领导突然发来的哪几个字？
```

这些不是简单互动句，而是通向 HIRO 的入口。

尤其这一句：

```text
你最想把哪句话原封不动丢出来？
```

它可以成为 HIRO 产品首页的核心母题。

## 6. 用户最终如何流向 HIRO

### 6.1 流量路径

未来用户路径应该是：

```text
用户刷到内容
↓
觉得“这说的是我”
↓
看到评论引导
↓
在评论区说出具体经历，或产生表达冲动
↓
看到轻承接话术
↓
点击/搜索/进入 HIRO
↓
把那句话丢给 HIRO
↓
HIRO 帮他整理情绪、处境、回应方式
↓
用户感到被接住
↓
进入更深层功能或付费服务
```

### 6.2 轻承接话术

核心话术是：

```text
如果你也卡在这儿，可以先把话丢给 HIRO。
```

这句话非常重要，因为它不是硬卖。

它没有说：

```text
立即购买
扫码咨询
领取资料
点击体验AI助手
```

而是说：

```text
你可以先把话丢过来。
```

这和用户的心理状态一致。

用户此刻不是来买产品的，他只是卡住了，不知道该怎么说。

HIRO 要承接的不是“购买意图”，而是“表达冲动”。

### 6.3 各平台承接方式

#### 小红书

小红书适合轻，不要太像广告。

正文里不一定直接放 HIRO。

可以通过评论区承接：

```text
如果你也卡在这儿，可以先把话丢给 HIRO。
```

或者：

```text
不用整理，先把那句话发出来。
```

#### 知乎

知乎适合更理性一点，但仍然不能像广告。

可以写：

```text
如果你也卡在这件事里，可以先试试把它写出来。
不用整理，不用体面，就把那句话原样写下来。
有时候说出来，就已经是好开始。
```

#### 公众号

公众号是自己的阵地，可以更明确承接：

```text
如果你也卡在这儿，可以先把话丢给 HIRO。
不用整理，不用讲得体面。
就把那句话发过来。
```

#### 短视频

短视频适合字幕式轻承接：

```text
如果你也卡在这儿，可以先把话丢给 HIRO。
```

## 7. HIRO 产品本体应该是什么

### 7.1 HIRO 首页定位

HIRO 首页不应该像传统 SaaS 官网。

不要一上来写：

```text
专业AI情绪陪伴工具
智能分析你的职场困境
帮你成长为更好的自己
```

这些都太像AI产品，也太像营销。

HIRO 首页应该像一个能接住人的地方。

第一屏可以是：

```text
把那句话丢给 HIRO

不用整理。
不用讲得体面。
就把它原样发过来。
```

输入框 placeholder：

```text
比如：领导说我不够主动，但我已经连续加班两周了。
```

按钮：

```text
先说出来
```

### 7.2 HIRO 的第一步不是分析，而是接住

用户输入一句话后，HIRO 不应该立刻输出长篇分析。

第一轮回复应该遵循：

```text
1. 复述用户真实处境
2. 点出卡住的地方
3. 不安慰
4. 不给大道理
5. 给一个可以继续说下去的问题
```

例如用户输入：

```text
领导说我不够主动，但我已经连续加班两周了。
```

HIRO 不应该回答：

```text
我理解你的感受。职场中我们经常会遇到不公平的评价，你要学会调整心态。
```

这是错误方向。

HIRO 应该回答：

```text
你不是没主动。
你是已经把时间交出去了，对方还想要你把态度也交出去。

这句话最刺人的地方，可能不是“不够主动”，而是他像没看见你那两周的加班。

他当时是在会议里说的，还是单独跟你说的？
```

这就是“接住”。

### 7.3 HIRO 的核心能力模块

未来 HIRO 产品本体可以分为几个能力：

#### 1. 原话接住

用户丢一句话，HIRO 先接住。

目标：让用户觉得“它真的听懂了”。

#### 2. 情绪命名

不是简单说“你很难过”，而是具体命名：

```text
这不是单纯焦虑，更像是被否定后的慌。
```

#### 3. 处境拆解

把事情拆清楚：

```text
这里有三层：
1. 事实：你已经连续加班两周
2. 对方评价：你不够主动
3. 真正刺痛你的地方：努力没有被看见
```

#### 4. 回应生成

帮用户生成不委屈自己的回复。

例如：

```text
我理解你对主动性的要求。
但过去两周我已经连续参与了几个紧急任务。
如果需要我在主动性上调整，我希望能具体到任务优先级，而不是只用“不够主动”来概括。
```

#### 5. 复盘记录

把用户多次输入沉淀成他的情绪/职场档案：

- 常见触发点
- 经常卡住的关系
- 不敢说出口的话
- 高频出现的工作场景
- 适合他的表达方式

#### 6. 内容反哺

匿名聚合用户高频问题，反哺内容系统：

```text
用户最近都卡在什么地方？
哪些话题更容易让人开口？
哪些评论引导带来更多输入？
```

## 8. HIRO 的语言风格规范

HIRO 产品本体必须和外部获客内容保持一致。

### 8.1 必须保持的风格

```text
真实
具体
克制
像人话
能说中用户的卡点
不急着教育用户
不急着安慰用户
不急着卖产品
```

### 8.2 禁止风格

HIRO 内部回复禁止出现：

```text
我理解你的感受
你要相信自己
不要太焦虑
一切都会过去
你值得更好的
学会爱自己
停止内耗
和自己和解
提升认知
建立边界感
```

这些词会让用户觉得：

```text
外面内容像真人，里面产品像AI客服。
```

这会直接破坏信任。

### 8.3 HIRO 的一句核心原则

> 外面怎么说人话，里面也必须怎么说人话。

如果获客内容靠真实感把人吸引进来，产品内部就不能用模板话术把人推走。

## 9. 如何从用户转化为消费

### 9.1 转化不是硬卖，而是从“被接住”开始

用户不会因为看到一个AI工具就付费。

用户会因为下面这个体验付费：

```text
我刚才那句话，它真的听懂了。
```

所以 HIRO 的转化逻辑不是：

```text
展示功能 → 用户购买
```

而是：

```text
用户说出一句卡住的话 → HIRO接住 → 用户继续说 → HIRO帮他整理 → 用户发现有用 → 愿意为更深层服务付费
```

### 9.2 免费层应该给什么

免费层要给用户一次明确的“被接住”体验。

建议免费功能：

```text
每天免费 3 次“把话丢给 HIRO”
每次可以获得：
1. 一段接住式回应
2. 一个卡点拆解
3. 一个继续说下去的问题
```

不要免费给太多完整解决方案。

免费层目标：

> 让用户知道 HIRO 不是鸡汤，不是AI客服，而是真的能帮他把话说清楚。

### 9.3 付费层可以卖什么

付费不应该叫“高级会员”那么泛。

可以围绕真实场景设计：

#### 1. 回应生成包

用户最刚需的是“不知道怎么回”。

付费功能：

```text
帮我回这句话
帮我拒绝这件事
帮我向领导解释但不显得卑微
帮我表达不满但不撕破脸
```

这是最容易转化的功能。

#### 2. 职场对话模拟

用户可以输入：

```text
领导要找我谈话
同事又把活推给我
面试官问我为什么裸辞
```

HIRO 模拟对话，帮用户提前练习。

#### 3. 情绪档案

长期记录用户高频卡点：

```text
你最近一个月最常卡在：不敢拒绝、被否定、怕麻烦别人。
```

这类功能适合月付。

#### 4. 深度整理报告

把多轮聊天整理成：

```text
这件事的事实
你的情绪
对方的问题
你真正想说的话
你可以怎么回应
```

可以作为单次付费。

#### 5. 面试/职场专项包

结合原有“面试官/HIRO”项目经验，可以做：

```text
面试前一晚稳定包
被领导约谈应对包
裸辞后解释包
试用期沟通包
不敢拒绝训练包
```

### 9.4 价格策略初稿

未来可考虑：

```text
免费：每天3次基础接住
9.9元：单次回应生成/一次深度整理
19.9元：职场沟通专项包
29.9-49.9元/月：月度情绪与回应助手
99元：面试/职场高压场景专项包
```

价格先低，重点验证付费意愿。

不要一开始就做很贵的会员。

## 10. 产品页面结构建议

### 10.1 首页

首页只做一件事：让用户输入那句话。

结构：

```text
主标题：把那句话丢给 HIRO
副标题：不用整理，不用讲得体面。就把它原样发过来。
输入框：比如：领导说我不够主动，但我已经连续加班两周了。
按钮：先说出来
下方小字：HIRO 不会上课，也不劝你“想开点”。
```

### 10.2 结果页

用户提交后，结果页分三段：

```text
1. 它刺到你的地方
2. 这件事可以怎么拆
3. 你可以怎么说
```

不要用太专业的标题。

不要写：

```text
情绪分析
心理评估
认知重构
行动建议
```

这些会让产品变冷。

### 10.3 继续对话页

按钮文案：

```text
我还想说一句
帮我把这句话回过去
帮我说得不那么委屈
帮我整理成一段话
```

### 10.4 付费弹窗

不要写：

```text
升级会员解锁高级功能
```

可以写：

```text
这件事可以继续往下捋。

HIRO 可以帮你把：
- 想说但没说出口的话
- 不想撕破脸的回应
- 明天要发出去的那段消息
整理成能发出去的版本。
```

按钮：

```text
继续把这件事捋完
```

## 11. 数据闭环

未来必须追踪完整链路：

```text
内容话题
↓
平台
↓
标题点击/阅读
↓
收藏/评论
↓
评论引导类型
↓
进入 HIRO 人数
↓
输入第一句话人数
↓
完成首次回应人数
↓
继续对话人数
↓
付费人数
```

### 11.1 核心指标

最重要的不是播放量，而是：

```text
多少人愿意把那句话丢出来？
```

核心指标：

- 内容停留率
- 评论率
- 具体评论率
- HIRO 入口点击率
- 首句输入率
- 首次回应完成率
- 二次输入率
- 付费转化率

### 11.2 评论质量指标

评论不是越多越好。

更重要的是具体程度。

高质量评论特征：

```text
出现原话
出现时间点
出现具体人物
出现动作
出现场景
出现“我当时……”
```

低质量评论：

```text
太真实了
狠狠共鸣
说到心坎了
```

这些也有价值，但不如具体经历有价值。

## 13. 前后一致性原则

这是整个项目最重要的原则。

### 13.1 外部内容和内部产品必须同一种气质

外部内容如果是：

```text
真实、具体、克制、不鸡汤
```

内部产品也必须是：

```text
真实、具体、克制、不鸡汤
```

不能外面写：

```text
键盘敲得再快，锅还是你的。
```

进来之后变成：

```text
亲爱的用户，我理解你的困扰。请相信每一次经历都是成长。
```

这是灾难。

### 13.2 HIRO 永远不要站到用户对面

HIRO 不是老师，不是老板，不是咨询师架子。

HIRO 不说：

```text
你应该怎么做
你要调整心态
你也要反思自己
```

HIRO 要先说：

```text
这句话最难受的地方，是他没看见你已经做了什么。
```

然后再慢慢帮用户找下一步。

### 13.3 不把用户推向极端

HIRO 也不能煽动用户：

```text
马上辞职
直接怼回去
不要忍
彻底拉黑
```

HIRO 的正确位置是：

```text
不替用户忍，也不替用户冲。
帮用户把话说清楚。
```

## 14. HIRO 的产品人格

如果把 HIRO 当作一个人，它应该像：

```text
一个坐在你旁边的人。
不是老师。
不是心理专家。
不是职场导师。
也不是只会说“抱抱你”的朋友。

它听完你那句话，能说：
“这句确实难听，因为它把你的努力全抹掉了。”

然后问：
“他是在什么场合说的？”
```

HIRO 的人格关键词：

```text
接住
具体
清醒
克制
站在人这边
不装专业
不制造依赖
帮人把话说清楚
```

## 15. 未来开发时必须遵守的检查清单

每次开发 HIRO 新功能前，都要问：

```text
1. 这个功能是在帮用户把话说出来吗？
2. 它有没有变成说教？
3. 它有没有变成鸡汤？
4. 它有没有像AI客服？
5. 它是否和外部内容风格一致？
6. 它是否能承接“如果你也卡在这儿，可以先把话丢给HIRO”？
7. 它是否能让用户继续说下一句？
8. 它是否有清晰的消费理由？
9. 它有没有过早推销？
10. 它有没有尊重用户真实处境？
```

如果有一个答案不对，就不要上线。

## 16. 最终愿景

HIRO 最终不是一个“AI陪聊工具”。

它应该成为一个职场和生活高压场景里的表达工具。

用户在这里可以：

```text
把不敢说的话先说出来
把委屈拆清楚
把乱掉的事理顺
把要发出去的话写得不卑微
把明天要面对的人先模拟一遍
```

HIRO 不替用户做决定。

HIRO 帮用户把话拿回来。

项目最终要形成闭环：

```text
爬虫找到真实痛点
内容说出真实痛点
评论引导用户说出自己的真实痛点
HIRO 接住这个真实痛点
产品帮用户把真实痛点转成能行动的一句话
用户为这个“能说出口”的能力付费
```

这就是 HIRO 的核心价值。
